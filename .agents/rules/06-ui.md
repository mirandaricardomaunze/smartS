# UI Components & Styling

## Activation: Glob
## Pattern: src/components/**/*.tsx, src/features/**/components/**/*.tsx

## Styling Rules
- Use NativeWind (Tailwind classes) for all styles — no StyleSheet
- Explicit margins only: `mt-`, `mb-`, `mx-`, `my-` — never use `gap`
- Dark mode support via NativeWind `dark:` prefix
- Consistent border radius: `rounded-lg` for cards, `rounded-full` for badges

## Required Base Components
```
src/components/ui/
├ Button.tsx        → variants: primary, secondary, danger, ghost
├ Input.tsx         → with label, error message, optional icon
├ Card.tsx          → container with shadow and border radius
├ Badge.tsx         → status: success, warning, danger, info
├ Loading.tsx       → full screen and inline spinner
├ EmptyState.tsx    → icon + message for empty lists
├ Avatar.tsx        → user profile image with fallback initials
└ ConfirmDialog.tsx → confirmation modal before destructive actions
```

## Component Rules
- Always define TypeScript props interface
- Always handle: loading state, error state, empty state
- `accessibilityLabel` required on all interactive elements
- No business logic inside UI components
- Default export only

## Premium Aesthetics (WOW Factor)
- **Glassmorphism**: Use `Card` with `variant="glass"` for dashboard metrics and floating elements.
- **Shadows**: Use premium shadow tokens (`shadow-premium-sm` to `shadow-premium-lg`) instead of standard Tailwind shadows.
- **Gradients**: Use `LinearGradient` for headers and primary cards to create depth.
- **No Placeholders**: Never use generic icons or colors. Always use thematic icons (Lucide) and harmonious color palettes (Indigo/Slate).

## Form Layout & Spacing
- **Vertical Stacking**: Mandatory vertical layout for Phone and Email fields to ensure readability.
- **Field Labels**: Use `Nuit ou NIF` or `NIF / NUIT / Contribuinte` for tax identification fields.
- **Input Height**: Standard height for inputs should be `h-14` for a premium touch feel.
- **Bottom Margins**: All primary action buttons at the bottom of forms must have `mb-8` or equivalent padding to avoid being cut off by the home indicator.

## Premium Bottom Sheet Pattern
- **Mandatory Usage**: All creation, editing, and selection forms must use the custom `BottomSheet` component.
- **Animation**: Must animate from the bottom, originating from the tab bar area.
- **Gestures**: Swipe-to-close must be restricted to the "handle" area to avoid conflicts with scrollable content.
- **Aesthetics**: Use uniform background colors (bg-white / dark:bg-slate-950), eliminate heavy borders, and use themed handle colors (sky-500/10).
- **Placement**: Forms should appear above the current screen, maintaining the application context via background dimming.
